# SNAD Color System

> **Brand colors:** `#0E3D38` (Dark Petroleum Green) + `#D4AF37` (Royal Polished Gold)
> **Contrast target:** WCAG 2.2 AA (4.5:1 body, 3:1 large text)
> **Theme support:** Light (default) + Dark + OS preference fallback

The SNAD color system is **token-first**: every color in the UI is a
semantic token reference, never a raw value. This makes theme switching
trivial and brand drift impossible.

## 1. The brand palette

| Role | Name | Hex | RGB | WCAG AA on white | WCAG AA on petroleum |
| --- | --- | --- | --- | --- | --- |
| Primary | Dark Petroleum Green | `#0E3D38` | 14, 61, 56 | 11.2:1 ✓ | — |
| Accent | Royal Polished Gold | `#D4AF37` | 212, 175, 55 | 3.4:1 ✗ (large only) | 5.7:1 ✓ |

**Rules:**

* Petroleum green is the **primary** brand color. Use it for CTAs, brand
  text, app headers, and key brand moments.
* Gold is the **accent** brand color. Use it sparingly for awards, premium
  markers, and decorative emphasis.
* Gold **must not** be used as body text on light backgrounds — it fails
  WCAG AA (3.4:1 < 4.5:1).
* Gold **may** be used as text on petroleum backgrounds (5.7:1 ✓) or as a
  background fill with petroleum text on top (5.7:1 ✓).

black (600–950).

## 3. Semantic mapping

Raw scale steps are **never** used directly in components. They are mapped
to semantic tokens that describe *intent*, not *appearance*:

```css
/* ❌ Never do this — it bypasses theming */
.button { background: var(--snad-color-petroleum-700); }

/* ✅ Always use semantic tokens */
.button { background: var(--snad-color-action-primary); }
.button:hover { background: var(--snad-color-action-primary-hover); }
```

The semantic layer is what makes theme switching work: in dark mode, the
*same* `--snad-color-action-primary` token resolves to a different
petroleum scale step (lightened for dark surfaces).

### Background → Surface hierarchy

```
background-default          ← page canvas (ivory in light, near-black in dark)
  └─ background-elevated    ← raised section (white in light, deep teal in dark)
      └─ surface-primary    ← card / panel surface
          └─ surface-secondary ← nested element (warm gray in light, deep teal in dark)
```

Always nest surfaces in this order. A `surface-secondary` placed directly
on a `background-default` (skipping `surface-primary`) breaks the visual
hierarchy and confuses readers.

### Text hierarchy

```
text-primary                ← headings, body copy (charcoal in light, near-white in dark)
  └─ text-secondary         ← captions, supporting text (warm gray)
      └─ text-muted         ← placeholders, disabled (neutral gray)
text-inverse                ← text on brand-colored backgrounds (always opposite of text-primary)
text-brand                  ← brand-colored emphasis text (petroleum in light, gold-teal in dark)
```

## 4. Status colors

| Status | Foreground token | Soft background token | Use |
| --- | --- | --- | --- |
| Success | `--snad-color-success` | `--snad-color-success-soft` | Valid form, completed action |
| Warning | `--snad-color-warning` | `--snad-color-warning-soft` | Pending action, non-blocking warning |
| Error | `--snad-color-error` | `--snad-color-error-soft` | Invalid form, failed action |
| Info | `--snad-color-info` | `--snad-color-info-soft` | Informational banner |
| Pending | `--snad-color-pending` | (use warning-soft) | Awaiting external action |

**Rule:** status colors are always paired — a status message uses the
`soft` background and the matching foreground color. Never use a status
foreground color on a different status's soft background.

## 5. Dark mode rules

### What changes

1. **Brand colors are lightened** so they remain visible against dark
   surfaces:
   * Petroleum `#0E3D38` → `#1B6E66` (lightened ~40%)
   * Gold `#D4AF37` → `#E1C266` (lightened ~15%)
2. **Surfaces invert**: ivory → near-black, white → deep teal.
3. **Text inverts**: charcoal → near-white, warm gray → muted teal.
4. **Shadows darken** (pure black, 30–60% opacity) — light-tinted shadows
   disappear on dark backgrounds.
5. **Soft status backgrounds** invert (light tint → deep tint of the same
   hue).

### What does NOT change

1. The petroleum and gold **scales** themselves (50–950 hex values) —
   these are constants. Only the semantic tokens that *reference* the
   scales flip to a different step in dark mode.
2. The brand **identity** — petroleum green is still petroleum green, just
   lightened for accessibility.

### Theme switching

```tsx
// In a server component or layout:
<html data-theme={userPreference ?? "light"}>
  <body>{children}</body>
</html>
```

If no `data-theme` attribute is set, the `@media (prefers-color-scheme: dark)`
fallback in `theme.css` honors the user's OS preference.

## 6. Translucent overlays

For translucent brand-color tints (hover backgrounds, focus rings, soft
borders), use `color-mix()` with a token reference:

```css
/* ✅ Correct — uses token, theme-aware */
.hover-bg {
  background: color-mix(in srgb, var(--snad-color-brand-primary) 4%, transparent);
}

/* ❌ Wrong — hardcoded rgba, fails lint, doesn't adapt to theme */
.hover-bg {
  background: rgba(14, 61, 56, 0.04);
}
```

`color-mix()` is supported in all modern browsers (Chrome 111+, Safari
16.4+, Firefox 113+). The SNAD web app targets these browsers.

## 7. Contrast audit

All semantic token pairs were audited against WCAG 2.2 AA:

| Foreground | Background | Ratio | Pass |
| --- | --- | --- | --- |
| `text-primary` on `background-default` (light) | 16.4:1 | ✓ AAA |
| `text-primary` on `surface-primary` (light) | 17.9:1 | ✓ AAA |
| `text-secondary` on `background-default` (light) | 8.6:1 | ✓ AAA |
| `text-muted` on `background-default` (light) | 3.9:1 | ✗ — use only for non-essential text |
| `text-inverse` on `action-primary` (light) | 11.2:1 | ✓ AAA |
| `text-inverse` on `action-primary` (dark) | 7.4:1 | ✓ AAA |
| `action-primary` text on `background-default` (light) | 11.2:1 | ✓ AAA |
| `success` on `success-soft` (light) | 4.8:1 | ✓ AA |
| `error` on `error-soft` (light) | 4.7:1 | ✓ AA |
| `warning` on `warning-soft` (light) | 5.1:1 | ✓ AA |
| `info` on `info-soft` (light) | 4.6:1 | ✓ AA |

**`text-muted` exception:** the muted token (3.9:1 on default background)
falls below AA. It may be used **only** for:

* Placeholder text inside inputs
* Timestamps on cards
* Disabled control labels

It must **not** be used for body copy, captions that carry essential
information, or any text the user is expected to read.

## 8. Prohibited color usage

1. ❌ Hardcoded hex / rgb / rgba / hsl values anywhere except
   `design-system/tokens/theme.css`, `tokens.json`, and the legacy
   `snad-tokens.css` shim.
2. ❌ Direct use of scale-step tokens (`--snad-color-petroleum-700`) in
   components — always go through the semantic layer.
3. ❌ Gold as body text on light backgrounds.
4. ❌ Petroleum as body text on light backgrounds (use `text-primary`
   instead; petroleum is reserved for brand emphasis).
5. ❌ Mixing status colors (e.g. error foreground on warning-soft
   background).
6. ❌ Using `text-muted` for essential copy.
7. ❌ Adding new brand colors. The palette is closed: petroleum, gold, and
   the soft / deep / sage supporting tints. Nothing else.
