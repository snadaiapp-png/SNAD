# Stage 28 — First Paid Customer Conversion

## Purpose

This document defines the conversion path from qualified pilot/customer opportunity to first paid customer readiness.

The document prepares conversion governance and commercial execution. It does not activate billing or collect live payments.

## Conversion Definition

A first paid customer conversion is achieved when a qualified customer is ready to move from pilot or pre-commercial evaluation into a paid subscription or paid implementation arrangement under approved commercial, legal, tax, support, and billing controls.

## Conversion Entry Criteria

```text
Customer qualified
Pilot or value review completed
Executive sponsor confirmed
Paid scope documented
Commercial proposal prepared
Legal/commercial review path active
Billing approval gate defined
Support model ready
No unresolved critical security blocker
```

## Conversion Stages

```text
Qualified Customer
Pilot Value Review
Paid Fit Review
Commercial Proposal Draft
Commercial Approval
Legal/Tax Review
Billing Activation Approval Gate
Customer Acceptance
Paid Conversion Ready
```

## Customer Decision Checklist

| Decision Area | Required Evidence |
|---|---|
| Business value | Confirmed pain solved or measurable value |
| Scope | Modules, users, workflows, and support scope |
| Commercial | Proposal and pricing path |
| Legal | Review status and blockers |
| Billing | Separate approval required before activation |
| Customer success | Handoff and success owner |

## Paid Offer Components

```text
Subscription plan
Tenant/user limits
Included modules
Workflow volume assumption
Support model
Partner implementation scope
Onboarding scope
Billing cycle
Renewal period
Exclusions
```

## Approval Path

```text
1. Customer success value review
2. Commercial owner review
3. Legal/tax readiness review
4. Billing gate review
5. Owner approval
6. Customer acceptance
7. Paid conversion readiness record
```

## Conversion Risk Register

| Risk | Severity | Control |
|---|---|---|
| Customer value not confirmed | High | Value review required |
| Pricing not approved | Medium | Commercial approval gate |
| Legal blocker | High | Review before external commitment |
| Billing activated prematurely | High | Separate billing approval required |
| Support capacity gap | Medium | Customer success readiness gate |
