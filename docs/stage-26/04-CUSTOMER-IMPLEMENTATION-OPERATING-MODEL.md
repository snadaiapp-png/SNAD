# Stage 26 — Customer Implementation Operating Model

## Purpose

This document defines the operating model for implementing SNAD with first customers and early enterprise accounts. It aligns sales promises, technical delivery, customer success, support readiness, and operational governance.

## Implementation Principles

```text
- Start with controlled scope
- Use pilot milestones
- Avoid unsupported custom delivery
- Preserve tenant isolation
- Keep customer data protected
- Capture feedback without disrupting production
- Require explicit approvals for high-impact workflows
```

## Implementation Phases

### Phase 1 — Readiness Review

Validate customer profile, legal readiness, data readiness, user count, organizational structure, required modules, and integration needs.

### Phase 2 — Tenant Setup

Prepare tenant configuration, admin users, roles, permissions, language settings, branding defaults, and initial workspace state.

### Phase 3 — Workflow Configuration

Document required workflows, approval paths, notifications, escalation rules, and audit requirements.

### Phase 4 — Data Preparation

Define data fields, import formats, data owners, validation rules, privacy obligations, and migration constraints.

### Phase 5 — User Enablement

Prepare onboarding sessions, role-based training, admin orientation, support routes, and knowledge-base references.

### Phase 6 — Go-Live Readiness

Validate production health, user access, support coverage, rollback path, issue escalation, and success metrics.

### Phase 7 — Post-Go-Live Support

Monitor adoption, incidents, questions, workflow usage, customer health, and improvement backlog.

## Implementation Artifacts

```text
- Customer readiness checklist
- Tenant setup checklist
- Role and permission matrix
- Workflow configuration sheet
- Data preparation sheet
- Training plan
- Go-live checklist
- Post-go-live support register
```

## Customer Success Handoff

The implementation team must hand over:

```text
Customer objectives
Configured modules
Known gaps
Open risks
Training completion
Support contacts
Success metrics
Renewal or expansion signals
```

## Implementation KPIs

```text
Time to tenant setup
Time to first admin login
Time to first workflow configured
Training completion rate
Go-live issues
Support tickets in first 14 days
Activation score
Customer health score
```

## Risks

```text
- Customer expects unsupported customization
- Data quality delays onboarding
- Missing legal approval delays activation
- Role permissions misconfigured
- Workflow scope expands without governance
- Support load exceeds capacity
```

## Acceptance Criteria

```text
Implementation phases documented
Artifacts listed
Customer success handoff defined
Implementation KPIs defined
Risks documented
Production remains LIVE
```
