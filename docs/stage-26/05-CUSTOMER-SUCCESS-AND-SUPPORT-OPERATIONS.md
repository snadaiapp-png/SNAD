# Stage 26 — Customer Success and Support Operations

## Purpose

This document defines the customer success and support operations model for SNAD as an operating company.

The model ensures that customers can be onboarded, supported, monitored, renewed, and expanded through governed operating processes.

## Customer Success Lifecycle

```text
Pre-Onboarding
Kickoff
Configuration
Training
Activation
Adoption
Health Monitoring
Quarterly Business Review
Renewal
Expansion
```

## Customer Success Responsibilities

| Responsibility | Description |
|---|---|
| Onboarding | Coordinate tenant setup, users, roles, workflows |
| Adoption | Track usage and business process adoption |
| Health Score | Monitor customer risk and growth signals |
| Retention | Manage renewal readiness and churn prevention |
| Expansion | Identify module, user, and partner expansion |
| Feedback | Route product feedback into roadmap governance |

## Health Score Model

The initial health score uses a 0–100 scale:

| Signal | Weight |
|---|---:|
| Login/activity | 20 |
| Feature adoption | 20 |
| Workflow usage | 15 |
| Support load | 15 |
| Executive engagement | 10 |
| Renewal readiness | 10 |
| Expansion signal | 10 |

## Support Model

Support tiers:

```text
Severity 1: Production unavailable or critical customer impact
Severity 2: Major workflow or operational disruption
Severity 3: Standard product issue or configuration blocker
Severity 4: Question, enhancement, or documentation request
```

## Escalation Rules

```text
Severity 1: Immediate escalation to owner/engineering
Severity 2: Same-day triage
Severity 3: Planned support response
Severity 4: Backlog/documentation routing
```

## Support Channels

Initial supported channels:

- GitHub issue tracking for internal execution
- Email/helpdesk readiness
- Customer success contact
- Partner escalation path
- Future in-app support channel
