## Legend

- **Typed**: ✅ = returns a typed DTO; ❌ = returns `Map<String, Object>` (raw row)
- **Pagination**: ✅ = supports `limit/cursor/sort/direction`; ❌ = unbounded / `limit` only
- **Concurrency**: ✅ = emits `ETag` on GET + validates `If-Match` on PATCH; ❌ = neither
- **Idempotency**: ✅ = honors `Idempotency-Key` header; ❌ = no replay protection
- **OpenAPI**: ✅ = schema generated by springdoc; ❌ = untyped, no schema

## Patterns flagged for elimination

The following anti-patterns were present in the v1 controller and are
forbidden in v2 (enforced by `scripts/crm/api-contract-governance-check.sh`):

- `Map<String, Object>` as a public controller return type (44 occurrences)
- `List<Map<String, Object>>` as a public list return type (14 occurrences)
- `SELECT *` in repository queries (4 occurrences in `CrmService.one()` + `ensureDefaultPipeline()`)
- `queryForList` returning raw Maps (12 occurrences across `CrmService` + `CrmExtendedService`)
- Missing `ETag` on single-record GETs (8 endpoints)
- Missing `If-Match` validation on PATCH (6 endpoints)
- Missing `Idempotency-Key` support on POST (8 endpoints)
- Unbounded list endpoints (default limit 50, max 200, no cursor) — 14 endpoints
- No standard error envelope (errors use Spring's `ResponseStatusException`)

## Frontend consumer

All v1 endpoints are consumed by `apps/web/lib/api/crm.ts`. The v2 migration
of the frontend client is tracked in section 17 of the EXEC-PROMPT-CRM-003
brief; the v1 client remains unchanged so CRM-G1 functionality is preserved.
