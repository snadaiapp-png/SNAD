# Stage 18 — Customer Success Scale Model

**Date**: 2026-07-08

---

## Customer Success Framework

### Onboarding Phase (Week 1-2)

```
Day 1:
  - Welcome email (bilingual)
  - Login credentials
  - Onboarding checklist
  - Product tour (interactive)

Day 2-3:
  - Account setup (tenant, users, roles)
  - Initial data import (if applicable)
  - First feature use (CRM or workspace)

Day 4-7:
  - Feature adoption tracking
  - Check-in email (usage, questions)
  - Training session (30 min, on-demand)

Day 8-14:
  - Second feature adoption
  - Health score calculation
  - Feedback collection
  - Activation check (first value action)
```

### Activation Phase (Week 3-4)

```
Activation criteria:
  - User completed login
  - Accessed workspace
  - Used at least 1 core feature (CRM, task, etc.)
  - Returned within 7 days (not one-time use)

Activation target: > 60% within 7 days

If not activated:
  - Email outreach (day 7)
  - Product walkthrough offer (day 10)
  - Account manager call (day 14)
  - Re-engagement campaign (day 21)
```

### Adoption Phase (Month 2-3)

```
Adoption tracking:
  - Daily active users (DAU)
  - Weekly active users (WAU)
  - Feature usage breadth (how many features used)
  - Feature usage depth (how often)

Adoption targets:
  - > 40% of users active weekly
  - > 30% using CRM module
  - > 20% using Control Plane (admins)
  - > 50% using bilingual features (at least once)

Adoption interventions:
  - Low usage: Training offer
  - Single feature: Expand to other features
  - Admin not using Control Plane: Admin training
  - Language not switched: Highlight bilingual benefit
```

### Health Score

```
Health score components (0-100):
  - Usage frequency (30%): DAU/WAU trend
  - Feature adoption (20%): Breadth + depth
  - Support sentiment (15%): Ticket satisfaction
  - Growth signals (15%): User additions, module adds
  - Engagement (10%): Login frequency, session length
  - Payment health (10%): On-time payments

Health tiers:
  - 80-100: GOOD (healthy, expansion candidate)
  - 60-79: OK (stable, monitor)
  - 40-59: WARNING (at risk, intervention needed)
  - 0-39: CRITICAL (churn risk, immediate action)

Calculation: Weekly (automated)
Alerts: WARNING → account manager, CRITICAL → owner
```

### Quarterly Business Review (QBR)

```
Participants: Customer stakeholders + SNAD account manager
Duration: 60-90 minutes
Frequency: Quarterly (enterprise), Semi-annual (professional)

Agenda:
  1. Review period (usage, achievements, challenges)
  2. Health score review
  3. ROI discussion (value delivered)
  4. Roadmap alignment (what's coming)
  5. Feedback collection
  6. Expansion discussion (upsell)
  7. Action items

Deliverables:
  - QBR deck (customized per customer)
  - Action item list
  - Follow-up timeline

Target: 100% of enterprise customers receive QBR
```

### Support Escalation

```
Support tiers:
  Tier 1: Self-service (knowledge base, docs)
  Tier 2: Email support (24h response professional, 4h enterprise)
  Tier 3: Live chat (business hours, enterprise only)
  Tier 4: Phone support (enterprise only, critical issues)

Escalation triggers:
  - Critical issue: Immediate Tier 4
  - High issue unresolved > 24h: Escalate to Tier 3
  - Medium issue unresolved > 3 days: Escalate to Tier 2
  - Customer dissatisfaction: Account manager + owner

Escalation path:
  Support agent → Senior support → Engineering → Owner
```

### Renewal Management

```
Renewal timeline (annual contracts):
  Day -90: Renewal notification
  Day -60: Account manager outreach
  Day -30: Renewal proposal
  Day -14: Follow-up
  Day -7: Final reminder
  Day 0: Renewal or expiration

Renewal health check:
  - Health score > 60: Likely to renew
  - Health score 40-59: At risk, intervention needed
  - Health score < 40: High churn risk, executive engagement

Renewal targets:
  - Enterprise: > 95% renewal rate
  - Professional: > 85% renewal rate
  - Overall: > 90% renewal rate
```

### Expansion Management

```
Expansion triggers:
  - User count > 80% of limit
  - Feature requests for higher tier
  - Health score > 80 (healthy, ready to expand)
  - Customer-initiated upgrade request

Expansion process:
  1. Account manager identifies opportunity
  2. Value proposition prepared (ROI calculation)
  3. Customer outreach (demo of new features)
  4. Quote and proposal
  5. Contract amendment
  6. Feature activation
  7. Training (if needed)

Expansion targets:
  - > 30% of customers expand within first year
  - > 50% of enterprise customers expand within 2 years
```
