# SNAD API Standard

## Version Prefix
All API paths use `/api/v1` prefix.

## Path Conventions
- Plural nouns for resources: `/api/v1/users`, `/api/v1/organizations`
- lowercase, kebab-case when needed
- No verbs in CRUD paths
- Sub-resources only for clear ownership: `/api/v1/organizations/{orgId}/memberships`
- No Tenant ID in path — extracted securely from JWT

## HTTP Status Codes
| Status | Usage |
|---|---|
| 200 | GET success |
| 201 | POST created resource |
| 204 | DELETE without body |
| 400 | Validation error |
| 401 | Authentication required/invalid |
| 403 | Access denied |
| 404 | Resource not found |
| 409 | Resource conflict |
| 422 | Business validation failure |
| 429 | Rate limit exceeded |
| 500 | Unexpected server error |

## Content Type
- Request: `application/json`
- Error response: `application/problem+json`

## Request ID
- Header: `X-Request-Id`
- Generated if not provided (UUID)
- Returned in all responses
- Included in error body

## Pagination
- Parameters: `page` (0-based), `size` (1-100), `sort` (field,direction)
- Defaults: page=0, size=20
- Response: `PageResponse<T>` with `PageMetadata`

## Date/Time Format
- ISO-8601 UTC with `Z` suffix
- Date only: `YYYY-MM-DD`

## Null Semantics
- `null`: Value unknown or not applicable
- Empty string: Known empty text (when allowed)
- Empty array: Known collection with zero items
- Missing field: Not returned by this contract

## Versioning Policy
- URI path versioning only: `/api/v1/`
- Non-breaking: Optional field additions, new endpoints
- Breaking: Field removal, field rename, type change, required field addition
