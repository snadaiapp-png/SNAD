```

2. Retail Chains (5-50 stores)

3. Manufacturing (small to mid-sized)
   Need: ERP, inventory, accounting, HRM
   Budget: $500-$5,000/month

4. Healthcare (clinics, small hospitals)
   Need: HRM, accounting, compliance
   Budget: $300-$1,500/month

5. Construction (mid-sized contractors)
   Need: Project management, accounting, HRM
   Budget: $500-$3,000/month
```

## Ideal Customer Profile (ICP)

```
Company size: 50-500 employees
Geography: Saudi Arabia / GCC
Industry: Professional services, retail, manufacturing
Annual revenue: $5M-$50M
IT maturity: Medium (has IT team or consultant)
Decision maker: CTO, CFO, or COO
Pain: Disconnected tools, manual reporting, no bilingual system
Budget: $500-$3,000/month for business software
Timeline: 1-3 months to purchase
```

## Business Pain Points

```
1. Disconnected Tools
   "We use 5 different systems that don't talk to each other."
   SNAD value: Unified platform (CRM, ERP, HRM, Accounting in one)

2. No Bilingual System
   "Our Arabic staff struggle with English-only software."
   SNAD value: Native Arabic (RTL) + English (LTR), 168 translation keys

3. Manual Reporting
   "I spend 2 days every month compiling reports from different systems."
   SNAD value: Real-time dashboards, automated reporting

4. Data Silos
   "Sales doesn't know what operations is doing."
   SNAD value: Cross-module automation, shared data

5. Slow Onboarding
   "New employees take weeks to learn our tools."
   SNAD value: Intuitive UI, bilingual, built-in training

6. Compliance Risk
   "We're not sure if we comply with Saudi data protection laws."
   SNAD value: PDPL-aware design, data residency options
```

## Enterprise Sales Cycle

```
Stage 1: Lead Generation (Week 1)
  - Identify target accounts (LinkedIn, referrals, events)
  - Research decision makers
  - Initial outreach (email, LinkedIn, phone)

Stage 2: Discovery Call (Week 2)
  - 30-45 min call with decision maker
  - Understand pain points
  - Qualify (budget, authority, need, timeline)
  - Schedule demo

Stage 3: Product Demo (Week 3)
  - 60 min live demo (bilingual, customized)
  - Show relevant modules (CRM for services, ERP for manufacturing)
  - Address specific pain points
  - Show AI features (with human oversight)
  - Offer 14-day trial

Stage 4: Trial Period (Week 4-5)
  - 14-day Professional tier trial
  - Onboarding support (1-hour setup session)
  - Mid-trial check-in (usage, feedback)
  - Address objections

Stage 5: Proposal (Week 6)
  - Custom quote based on needs
  - Include SLA, DPA (enterprise)
  - Address final objections
  - Present ROI calculation

Stage 6: Contract (Week 7-8)
  - Legal review (MSA, SLA, DPA)
  - Security questionnaire
  - Negotiation (price, terms, custom features)
  - Signature

Stage 7: Onboarding (Week 9-10)
  - Tenant creation
  - User setup and training
  - Data migration (if applicable)
  - Go-live

Total cycle: 8-10 weeks (enterprise), 2-4 weeks (professional)
```

## Deal Stages

```
1. Prospect — Identified, not contacted
2. Contacted — Initial outreach sent
3. Qualified — BANT confirmed (Budget, Authority, Need, Timeline)
4. Demo Scheduled — Discovery call complete, demo booked
5. Demo Complete — Product demonstrated, trial offered
6. Trial Active — 14-day trial in progress
7. Trial Complete — Trial ended, proposal sent
8. Negotiation — Contract terms discussed
9. Closed Won — Contract signed
10. Closed Lost — Deal lost (document reason)
```

## Customer Objections & Responses

```
Objection: "It's too expensive"
Response: "At $49/month for Professional tier, SNAD replaces 3-5
  separate tools that cost $200-$500/month combined. You save
  60-80% while getting bilingual support."

Objection: "We already use [competitor]"
Response: "SNAD is the only bilingual (Arabic RTL + English LTR)
  business operating system built for Saudi Arabia. Can your
  current tool switch languages instantly for your Arabic staff?"

Objection: "We need SSO"
Response: "SSO/SAML is on our enterprise roadmap (Stage 19-20).
  We support standard email/password auth now, with SSO coming
  for enterprise customers."

Objection: "Is our data safe?"
Response: "SNAD is PDPL-aware, with encrypted transit, RBAC,
  tenant isolation, and audit logging. We can discuss data
  residency options for your compliance needs."

Objection: "We need custom features"
Response: "Our Workflow Builder allows custom workflows without
  coding. For deeper customization, we offer professional services
  and have a Marketplace roadmap for third-party extensions."

Objection: "What about AI safety?"
Response: "All AI features require human confirmation for
  high-impact actions. AI can recommend, but humans decide.
  No autonomous financial, deletion, or permission actions."
```

## Demo Flow

```
1. Introduction (5 min)
   - SNAD overview (bilingual, unified, compliant)
   - Agenda review

2. Login & Language Switch (5 min)
   - Show Arabic RTL login screen
   - Switch to English LTR
   - Switch back to Arabic
   - Highlight: instant, no reload, persists

3. Theme Switch (3 min)
   - Show light/dark/system modes
   - Highlight: no FOUC, persists

4. Workspace Tour (10 min)
   - Dashboard overview
   - User info, tenant info, session status
   - ExecutiveShell (logo, switchers, navigation)

5. CRM Module (10 min)
   - Contacts, deals, pipeline
   - Show AI lead scoring (if available)
   - Highlight: permission-filtered, tenant-isolated

6. Control Plane (5 min)
   - User management
   - Role assignment
   - Tenant settings

7. AI Features (7 min)
   - Show AI recommendation (with explanation)
   - Highlight: human confirmation required
   - Show audit log

8. Cross-Module Workflow (5 min)
   - Show workflow builder (if available)
   - Show cross-module automation pattern

9. Q&A (10 min)
   - Address specific questions
   - Discuss pricing and tiers
   - Offer trial

Total: 60 minutes
```

## Pilot to Paid Conversion

```
Conversion criteria:
  1. Trial user completed at least 1 core action (CRM contact, deal, etc.)
  2. Trial user logged in at least 3 different days
  3. Trial user invited at least 1 team member
  4. Trial user used bilingual feature at least once
  5. Trial user did not report any blocking issues

Conversion process:
  Day 1: Welcome email + onboarding guide
  Day 3: Check-in email (usage, feedback)
  Day 7: Mid-trial review (usage, objections)
  Day 11: Upgrade reminder (3 days left)
  Day 13: Final reminder (1 day left)
  Day 14: Trial ends → offer paid plan
  Day 15+: Follow-up (why didn't you convert?)

Conversion target: 30% trial-to-paid
```
