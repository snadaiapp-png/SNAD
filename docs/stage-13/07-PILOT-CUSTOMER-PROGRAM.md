# Stage 13 — Pilot Customer Program

**Date**: 2026-07-08

---

## Pilot Program Objectives

```
1. Validate product-market fit
2. Gather real-world usage feedback
3. Identify bugs and UX improvements
4. Test bilingual (Arabic/English) experience
5. Build case studies for marketing
6. Transition pilots to paying customers
```

## Pilot Customer Selection Criteria

```
Requirements:
  - Organization size: 5-50 employees
  - Industry: Any (priority: professional services, retail, consulting)
  - Technical readiness: Basic computer literacy
  - Commitment: 30-90 days active usage
  - Feedback willingness: Weekly check-ins
  - Language: Arabic or English speakers

Ideal pilot profile:
  - Small business owner or operations manager
  - Needs business operating system (not just CRM)
  - Values bilingual support (Arabic primary)
  - Willing to provide detailed feedback
  - Open to case study participation
```

## Pilot Program Parameters

```
Duration: 90 days
Number of pilots: 3-5 customers
Cost: Free (Free Pilot tier)
Support: Weekly check-ins + GitHub issue support
Features: All Professional tier features
Data: Pilot data retained for 30 days after program end
```

## Success Metrics

```
Pilot success criteria:
  - Active usage: > 3 days per week per user
  - Feature adoption: CRM used by > 50% of pilot users
  - Bilingual usage: Both Arabic and English tested
  - NPS score: > 30
  - Feedback: At least 5 actionable items per pilot
  - Conversion: > 40% of pilots convert to paid

Program success criteria:
  - 3+ pilots complete the 90-day program
  - 2+ pilots convert to paid Professional tier
  - 1+ case study produced
  - 10+ product improvements implemented from feedback
```

## Feedback Collection

```
Weekly:
  - 30-minute check-in call
  - Usage report review
  - Open-ended feedback session

Monthly:
  - NPS survey
  - Feature satisfaction survey
  - UX walkthrough

End of pilot:
  - Comprehensive debrief
  - Case study interview
  - Transition planning (to paid or exit)
```

## Pilot Onboarding Process

```
1. Recruit: Identify potential pilots (network, referrals)
2. Qualify: Verify they meet selection criteria
3. Agreement: Sign Pilot Agreement (simplified)
4. Provision: Create tenant, invite users
5. Train: 1-hour onboarding session
6. Support: Weekly check-ins begin
7. Monitor: Track usage and feedback
8. Iterate: Implement improvements based on feedback
9. Convert: Offer paid plan at end of pilot
```
